# 面向智慧教育认知诊断的思维链可解释智能体

## 一、项目简介

EduCD-Agent 是一个面向智慧教育场景的认知诊断可解释智能体原型。

本项目围绕“学生答题数据如何转化为可解释学情报告”这一核心问题，构建了一个融合认知诊断模型与大语言模型的智能体系统。系统首先基于学生答题记录和题目知识点 Q 矩阵，利用 DINA 认知诊断模型计算学生在不同知识点上的掌握程度；随后调用 Qwen Api，对学生错题进行错因分析、思维断点定位和个性化学习建议生成；最后形成结构化学情报告，并在前端页面中进行可视化展示。

系统不仅支持学生个体诊断，还支持教师端班级诊断摘要展示，包括班级薄弱知识点、高风险学生列表和教学干预建议，形成从“答题数据 → 认知诊断 → 错因解释 → 学情报告 → 教学干预”的智慧教育闭环。

---

## 二、核心功能

### 1. 学生个体诊断

系统支持选择学生，完成以下分析：

- 读取学生答题记录；
- 计算学生知识点掌握度；
- 找出学生错题；
- 调用 Qwen 生成错因分析；
- 定位学生思维断点；
- 生成个性化学情报告；
- 在前端展示知识点掌握图表、错因分析卡片和完整报告。

### 2. 错因分析

系统对每道错题生成结构化错因分析，主要包括：

- 错因类型；
- 薄弱知识点；
- 思维断点；
- 错误原因；
- 学习建议；
- 置信度。

错因类型包括：

| 错因类型 | 含义 |
|---|---|
| ConceptError | 概念理解错误 |
| FormulaError | 公式使用错误 |
| CalculationError | 计算错误 |
| StepJumpError | 步骤跳跃 |
| TransferError | 知识迁移失败 |
| CarelessError | 审题粗心 |

### 3. 学情报告生成

系统会根据学生知识点掌握度和错因分析结果生成结构化中文学情报告，报告包括：

1. 总体表现；
2. 知识点掌握情况；
3. 主要薄弱点；
4. 典型错因分析；
5. 个性化学习建议；
6. 后续练习路径。

### 4. 教师班级看板

教师端支持查看班级整体情况，包括：

- 班级平均知识点掌握度；
- 班级薄弱知识点 Top3；
- 高风险学生列表；
- 教师教学建议。

## 三、技术架构

系统整体采用前后端分离架构。

```text
学生答题数据
    ↓
Q 矩阵与知识库
    ↓
简化 DINA 认知诊断模型
    ↓
知识点掌握度
    ↓
Qwen 错因分析 Agent
    ↓
Qwen 学情报告 Agent
    ↓
Reflection 自检 Agent
    ↓
Flask API
    ↓
Vue + Element Plus 前端展示
```

## 四、运行步骤
进入后端目录
cd EduCD-Agent/backend
安装依赖
pip install -r requirements.txt
配置 Qwen API Key
$env:DASHSCOPE_API_KEY="你的APIKey"
运行后端
python app.py
进入前端目录
cd EduCD-Agent/frontend
安装依赖
npm install
启动前端
npm run dev
